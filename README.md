# MOMIJI Terminal 🍁

**AI ネイティブなターミナルアプリ**（macOS）。Claude Code・Codex・Gemini などの AI CLI エージェントを、分割ペイン・タブ・テーマ・記憶ログなどと一緒に快適に使えます。

> [!IMPORTANT]
> **【初回起動の必読】「"MOMIJI.app" は壊れているため開けません」と表示されても、アプリは壊れていません。**
> 未公証アプリを Apple Silicon でダウンロードした際に出る macOS の標準警告です。
> **アプリを「アプリケーション」フォルダへ移動したあと**、「ターミナル」アプリで次のコマンドを実行してから起動してください：
> ```
> xattr -cr /Applications/MOMIJI.app
> ```
> 実行後は通常どおりダブルクリックで起動できます。※「右クリック →『開く』」では回避できないことが多いため、上のコマンドをお使いください。

## インストール手順

1. `MOMIJI_x.x.x_aarch64.dmg` をダウンロードして開く
2. 表示された **MOMIJI.app** を **アプリケーション** フォルダにドラッグ＆ドロップ
3. 初回起動時、macOS のセキュリティ警告が出る場合があります（下記参照）

### ⚠️「"MOMIJI.app" は壊れているため開けません」と出る場合（初回のみ・重要）

**アプリは壊れていません。** 本アプリは Apple の公証（Notarization）を受けていないため、ブラウザでダウンロードすると macOS が“隔離属性”を付け、特に Apple Silicon（M1〜M3 以降）では「壊れている」という表示になります。次の操作で開けます。

**✅ 確実な方法（推奨）**
1. **MOMIJI.app を「アプリケーション」フォルダへ移動**（DMG からドラッグ）
2. **「ターミナル」アプリ**を開き、次の1行を貼り付けて実行：

   ```bash
   xattr -cr /Applications/MOMIJI.app
   ```

3. その後、**MOMIJI.app をダブルクリック**で通常起動できます（以降は警告なし）

> `xattr -cr` は「ダウンロード由来」の隔離属性を消すコマンドで、アプリ本体は変更しません。
>
> ※「壊れている」の表示のときは **右クリック→「開く」では回避できないことが多い**ため、上記コマンドをお使いください。
